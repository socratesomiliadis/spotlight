# Spotlight Staff Guide

## 1. Finding a Creator Email

When you find a project worth submitting, first confirm the project is real, public, and belongs to the creator you are adding.

Good places to look for the creator's email:

- The creator's personal website, portfolio, or agency website
- Contact, About, Press, or Team pages
- The footer of the project website
- GitHub profile, README, or repository commit/contact info
- LinkedIn, X/Twitter, Instagram, Behance, Dribbble, Product Hunt, or other public profile links
- Public WHOIS/business records only when the email is intentionally listed for contact
- Newsletter, press kit, media kit, or launch announcement

Helpful places to check:

- [Google](https://www.google.com): search the project name, creator name, company name, and "contact"
- [LinkedIn](https://www.linkedin.com): check the creator/company profile
- [X/Twitter](https://x.com): check bios, pinned posts, and linked websites
- [GitHub](https://github.com): check profiles, repositories, and README files
- [Product Hunt](https://www.producthunt.com): useful for launched products and maker profiles
- [Behance](https://www.behance.net) and [Dribbble](https://dribbble.com): useful for design work
- [WHOIS](https://lookup.icann.org): use only if the email is publicly listed

Only use public, professional contact information. Do not guess private emails, use leaked data, buy contact lists, or contact unrelated personal addresses.

Before creating an unclaimed account, check:

- The email looks like it belongs to the creator or their team
- The name, username, avatar, and project all match the same person/company
- The creator does not already have a claimed Spotlight account
- The same project has not already been submitted

## 2. Creating an Unclaimed Account and Submitting a Project

Use the staff submission page: `/projects/new/staff`.

1. Sign in with a staff/admin account.
2. Open `/projects/new/staff`.
3. Under Project Owner, search for an existing unclaimed user first.
4. If no match exists, choose Create New.
5. Add the creator's display name, username, email, and profile picture.
6. Submit the unclaimed account.
7. Fill in the project details:
   - Project title
   - Category: Websites, Design, Films, Crypto, Startups, or AI
   - Project URL
   - Banner image
   - Main project image
   - Preview video, if available
   - Tags
   - Up to 4 element images
8. Create the project and review the public project page.

Keep submissions clean: use high-quality images, accurate titles, relevant tags, and the creator's real public identity.

Do not send the creator any temporary password. They should claim the account through their profile's Claim Account button or `/claim/[username]`, using the same email you added.

## 3. Curating Project Content

Before submitting, collect the best public materials for the project. The goal is to make the project look accurate, polished, and exciting without misrepresenting it.

What to capture:

- Banner image: a wide, clean image that gives a strong first impression
- Main project image: the best representative screenshot or visual
- Preview video: a short recording that shows the project in motion
- Element images: up to 4 extra screenshots showing important pages, details, features, or visual moments
- Tags: choose only relevant tags based on what the project actually uses or represents

For websites and apps:

- Open the live project in a clean browser window
- Use desktop screenshots unless the project is mainly mobile
- Capture the homepage or most visually representative screen for the main image
- Capture useful secondary screens for element images
- Avoid screenshots with browser UI, cursor, popups, cookie banners, broken layouts, or loading states
- If the site has animation or interaction, record a short preview video

For preview videos:

- Keep it short and focused, ideally 5-10 seconds
- Show the strongest interaction, transition, scroll, or product moment
- Avoid shaky mouse movement and unnecessary waiting
- Do not include private user data, admin areas, or anything that is not public
- Use the [Spotlight Video Tool](https://video-tool.spotlight.day) to prepare the preview video

For films, design, crypto, startups, and AI projects:

- Use official public images, launch materials, screenshots, trailers, demo clips, or press assets when available
- Prefer visuals from the creator's own website or public profiles
- Do not use copyrighted images from unrelated articles unless they are clearly official/public project assets
- If the work is visual, pick images that show the craft clearly
- If the work is technical, pick images that help visitors understand what it does

Quality checklist:

- Images are sharp and not distorted
- Cropping looks intentional
- The project title matches the original project
- The URL works
- Nothing sensitive or private is visible
- The submission feels like a fair representation of the creator's work

## 4. Giving Awards

Use the staff dashboard: `/dashboard`.

1. Open `/dashboard`.
2. Search by project title or creator.
3. Filter by category or award if needed.
4. Click an award button:
   - Of the Day
   - Of the Month
   - Of the Year
   - Honorable
5. Choose the award date, month, or year.
6. Save the award.

Active awards are highlighted. Clicking an active award removes it. Clicking the award date lets you update the date.

## 5. Messaging the Creator

Keep the message short, warm, and clear. Tell them what won, where to view it, how to claim the account, and the exact email address staff used when creating the unclaimed account.

Template:

Subject: Your project won a Spotlight award

Hi [Name],

We loved [Project Name] and selected it for [Award Name] on Spotlight.

You can view it here: [Project Link]

We created an unclaimed creator profile for you so the award is connected to the right person/team.

Claim link: [Claim Link]
Email to use in the claim form: [Email Used For Unclaimed Account]

Once claimed, you can update your profile, manage the project, and keep the award attached to your account.

Congrats,
The Spotlight Team

## 6. Final Checklist

Before sending the message:

- Project page loads correctly
- Creator name, username, avatar, and email are correct
- Images and preview video look good
- Category and tags are accurate
- Award type and date are correct
- Claim/profile link works
- The message includes the same email used to create the unclaimed account

After the creator claims the account, avoid editing their profile unless they ask for help.
